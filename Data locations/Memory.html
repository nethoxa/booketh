Memory

You can think of memory as an array of byte[]. You can read and write to it, not as calldata, which is just read-only. First, you need to know something (photo from Jean Cvllr):

  • From 0x00 to 0x3F -> scratch space for hashing functions
  • From 0x40 to 0x5F -> FREE MEMORY POINTER
  • From 0x60 to 0x7F -> meant to be zero permanently and used as the initial value for empty dynamic memory arrays

If you put an offset higher than 2^64 - 1, the transaction will revert. The following data and values are always in memory by default:

  • Function parameters of complex types
  • Local variables of complex types
  • Values returned from functions, regardless of their type
  • Any complex value type returned by a function must specify the keyword memory
  • By variables/values of complex types, we refer to variables such as struct, arrays, bytes and strings

These variables defined with the keyword memory will disappear once the function call ends. For reads from memory, we are limited to 32 bytes words. However, we can use either 1 byte or 32 bytes words in writes. The reason underneath is due to the opcodes MLOAD to read (32 bytes), MSTORE and MSTORE8 to write (32 bytes and 8 bytes respectively).

For MLOAD

the formula is explained as follows (again, from Jean Cvllr):

  • Us[0] is the top item on the stack
  • Us'[0] is the return item PUSHed on the stack
  • Um is the *v[i], that is, the content of memory at a given offset i
  1. Take the last top item on the stack Us[0]
  2. Use this value as the starting pointer to read in memory Um
  3. Read from this memory pointer Us[0] the next 31 bytes that follow (Us[0] + 31)

We see now that we can only read from memory 32 bytes words at a time with a single MLOAD opcode.
